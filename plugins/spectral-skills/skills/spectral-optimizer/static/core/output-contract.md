# Optimizer Output Contract

Write these files in the optimizer output directory:

- `optimizer_contract.json`: mode, status, protocol, output references
- `optimization_plan.json`: task profile and selection policy
- `candidate_space.json`: candidate preprocessing, feature, and modeling space
- `trial_manifest.csv`: planned trials with no performance claims
- `trial_results.csv`: validation/CV result table, written only when real trial
  scores are supplied or produced by the optimizer executor; test columns are
  not used
- `best_pipeline.json`: selected trial if validation/CV results were supplied
- `recommendation_report.md`: human-readable recommendation

Do not write transformed matrices, final model artifacts, split contracts, or
cleaned packages. If the optimizer executes trials, it may automatically create
per-trial preprocess/feature/modeling outputs under `trials/<trial_id>/`; the
modeling outputs must be generated by `spectral-modeling --evaluation-mode
validation_only` and must not contain test metrics.
